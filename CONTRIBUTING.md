# Contributing

Thank you for your interest in contributing to the AI Task Capsule Framework.

This project exists to improve the safety, predictability, and governance of AI-assisted software development workflows.

We welcome improvements that strengthen repository integrity, reduce architecture drift, improve auditability, and enhance human oversight.

---

## Guiding Principles

All contributions should align with the core principles of the framework:

1. Explicit Scope
2. Explicit Permissions
3. Explicit Stop Conditions
4. Human Approval Gates
5. Risk-Based Execution

If a proposed contribution weakens any of these principles, it is unlikely to be accepted.

---

## Before Submitting Changes

Please ensure:

* The change has a clearly defined purpose.
* The change improves framework usability, safety, or governance.
* The change does not introduce ambiguity into existing contracts.
* Documentation is updated when behavior changes.

---

## Pull Request Guidelines

When opening a Pull Request, please include:

### Problem Statement

What governance, safety, or workflow problem does this change address?

### Proposed Solution

Describe the implementation and expected outcome.

### Risk Assessment

Identify any risks introduced by the change.

### Compatibility

Explain whether existing Capsule users must modify their workflows.

---

## Accepted Contribution Types

Examples include:

* New Capsule templates
* Governance improvements
* Additional risk tier definitions
* Enhanced stop-condition frameworks
* CI/CD integration examples
* Documentation improvements
* Multi-agent governance strategies
* Real-world implementation guides

---

## Not Recommended

The following contribution types are generally discouraged:

* Repository-specific customizations
* Framework-breaking changes without migration guidance
* Removal of safety controls
* Ambiguous execution rules
* AI autonomy features that bypass human review

---

## Code of Conduct

Contributors are expected to:

* Be respectful
* Be constructive
* Prioritize safety over convenience
* Prefer clarity over complexity
* Document assumptions

---

## Reporting Issues

When reporting issues, please include:

* Framework version
* Expected behavior
* Actual behavior
* Reproduction steps
* Relevant screenshots or examples

---

## Philosophy

If there is uncertainty regarding repository safety, architecture integrity, or security:

STOP.

Do not guess.

Do not assume.

Escalate to a human reviewer.

Safety takes precedence over convenience.

---

Thank you for helping improve the AI Task Capsule Framework.
